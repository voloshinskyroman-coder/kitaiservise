# Архитектура интеллектуального определения товара (MVP)

Цель MVP — отказаться от собственной базы ТН ВЭД и сложных интеграций на первом этапе. Вместо этого построить гибкую архитектуру, которую в будущем можно будет улучшить без изменения логики приложения.

──────────────────────────────

ШАГ 1. ВЫБОР ТОВАРА

Экран:

Заголовок:
Что хотите привезти?

Подзаголовок:
Начните вводить название товара или вставьте ссылку на Alibaba, 1688, Taobao или Made-in-China.

Одно универсальное поле ввода.

Поддерживается три сценария:

1. Пользователь вводит название товара.

Например:

фут...

↓

Появляются подсказки:

• Футболка
• Футболка поло
• Футболка спортивная
• Футболка детская

2. Пользователь вставляет ссылку.

Поддерживаются:

• Alibaba
• 1688
• Taobao
• Made-in-China

Система автоматически извлекает:

• название товара
• описание
• характеристики

3. Если товара нет в базе

Показывается кнопка:

Использовать введенное название

Далее AI самостоятельно анализирует введенный текст.

──────────────────────────────

ШАГ 2. БАЗА ПОИСКА

Для поиска НЕ используется ТН ВЭД.

Используется обычный каталог товаров.

Источники:

• Google Product Taxonomy
• Категории Ozon
• Категории Wildberries
• Каталог Alibaba

Из этих источников создается единый словарь поиска.

В базе хранится:

• id
• название
• синонимы
• категория

Например

Футболка

синонимы

майка
t-shirt
tee

категория

Одежда

Эта база используется только для поиска и автодополнения.

──────────────────────────────

ШАГ 3. AI АНАЛИЗ

После выбора товара AI получает:

• название
• описание
• ссылку (если есть)

AI определяет:

• категорию товара
• примерный код ТН ВЭД
• уровень уверенности
• необходимые документы
• необходимые сертификаты
• необходимую маркировку

Например

Футболка

↓

Категория

Одежда

↓

Код ТН ВЭД

6109100000

↓

Уверенность

93%

↓

Необходимо

• ДС
• EAC
• Честный Знак

ВАЖНО

На MVP AI самостоятельно определяет код ТН ВЭД.

Без использования внешних API.

──────────────────────────────

ШАГ 4. ВЫБОР СПОСОБА ДОСТАВКИ

Пользователь самостоятельно выбирает:

• Официальная доставка с таможенным оформлением (белая доставка)

или

• Упрощенная доставка (карго)

AI НЕ принимает решение за пользователя.

──────────────────────────────

ШАГ 5. ДОКУМЕНТЫ И СЕРТИФИКАЦИЯ

После определения товара система автоматически отмечает необходимые документы.

Например

☑ Декларация соответствия

☑ EAC

☑ Честный Знак

☐ Сертификат соответствия

☐ СГР

☐ РУ

Вверху отображается сообщение:

Мы автоматически определили необходимые документы для выбранного товара.
При необходимости вы можете изменить выбор.

Пользователь может:

• снять галочки
• поставить дополнительные
• оставить как есть

Остальные документы НЕ скрываются.

Пользователь всегда видит полный список.

──────────────────────────────

ШАГ 6. РАСЧЕТ

Если выбрана официальная доставка (белая доставка):

учитываются

• логистика
• пошлина
• НДС
• оформление
• сертификация
• маркировка

Если выбрана упрощенная доставка (карго):

не рассчитываются

• пошлина
• НДС

Но продолжают учитываться

• сертификация
• маркировка
• дополнительные услуги

Поскольку клиент может заказать их отдельно.

──────────────────────────────

ШАГ 7. САМООБУЧЕНИЕ

После обработки заявки сохраняется:

• исходный запрос пользователя
• ссылка (если была)
• выбранный товар
• код ТН ВЭД, определенный AI
• код ТН ВЭД, подтвержденный логистом

Таким образом постепенно формируется собственная база знаний компании.

Со временем большинство популярных товаров будет определяться автоматически.

──────────────────────────────
