Сделай архитектуру квиза для Telegram Mini App / web app “Китай Сервис”.
Цель приложения: собрать заявку на логистические услуги из Китая, определить нужные услуги, собрать данные груза, сделать предварительный расчет и передать структурированную заявку логисту.
ВАЖНО:
Это не обычный линейный квиз.
Это дерево сценариев + модульная система услуг.
Пользователь не должен видеть внутренние названия “клиент 0/1/2/3”.
Внутри системы можно использовать clientType: 0/1/2/3.
Логика:
1. Welcome screen
Кнопка: “Начать расчет”
2. Первый вопрос:
“Вы уже возили товары из Китая?”
Варианты:
- Официальной доставкой с таможенным оформлением (белая доставка)
- Упрощенной доставкой (карго)
- Нет, первый раз
Ответ НЕ влияет на ветку квиза.
Ответ сохраняется для логиста и влияет на скоринг лида.
3. Главный вопрос:
“Что вам сейчас нужно?”
Карточки:
1) Полный цикл поставки
Подпись: “От поиска поставщика до доставки”
clientType = 0
2) Нашел товар
Подпись: “Нужна помощь с оплатой и доставкой”
clientType = 1
3) Товар уже куплен
Подпись: “Нужна доставка и оформление”
clientType = 2
4) Нужны отдельные услуги
Подпись: “Экспорт, документы или логистика”
clientType = 3
Далее ветки расходятся по clientType.

РАСЧЕТ
Расчет должен работать после каждого ответа.
Каждый ответ обновляет объект Shipment.
В конце показываем не точную цену, а диапазон.
Для карго:
- расчет по весу
- расчет по объему
- берется большее значение
Для белой доставки:
- доставка
- таможенная пошлина
- НДС
- оформление
- брокер
- сертификация, если нужна
- маркировка, если нужна
Показывать клиенту:
- предварительная стоимость
- срок доставки
- что входит в расчет
- какие данные нужно уточнить
- кнопка “Получить точный расчет”
